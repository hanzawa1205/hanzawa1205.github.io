序列化组件

  • 方法1： 根据models中的字段转换

  • 方法2：

from .models import Product
from .serializers import ProductSerializers
from rest_framework.views import APIView
from rest_framework.response import Response

class ProductView(APIView):
def get(self, request):
product_list = Product.objects.all()
ps = ProductSerializers(product_list, many=True)
return Response(ps.data)

def post(self,request):
    ps = ProductSerializers(data=request.data)
    print(ps.is_valid())
    if ps.is_valid():
        ps.save()
        return Response(ps.data)
    else:
        return Response(ps.errors)

Restful规范

Veröffentlicht am 2019-05-09

restful规范（10）

1、 根据method不同，进行不同的操作: GET/POST/PUT/DELETE/PATCH
2、 面向资源编程，例如：https://hanzawa1205.github.io/archives
3、 URL中体现版本，例如https://hanzawa1205.github.io/v1/archives
4、 URL中说明是接口：例如https://hanzawa1205.github.io/api/v1/archives，尽量少出现如api.hanzawa的形式，这样可能会出现跨域请求
5、 尽量使用较为安全的HTTPS
6、 响应时设置规范的状态码：200 300 400 500 301临时重定向 302永久重定向
7、 条件例如https://hanzawa1205.github.io/archives？page=1&size=10
8、 不同method对应不同返回值
9、 返回错误信息 例如ret = {'code': 100001, 'error': 'xxx错误'}

hexo博客搭建

步骤1： 下载git
步骤2： 下载node.js
步骤3： 在桌面右键启动Git Bash后输入命令:
npm install -g hexo
步骤4： 创建一个’hexo’文件夹，在该文件夹中右键启动Git Bash后输入指令：
hexo init

此时hexo的静态文件等都已经加载到文件夹中。其中_config.yml文件可以修改博客的配置(标题，主题等等)
启动博客hexo g、hexo s
写博客：路径：\Hexo\source_posts，指令：hexo new '博客名'

##部署到Github
步骤1： 在GitHub上创建仓库，仓库名为hanzawa1205.github.io
步骤2： 在_config.yml中配置

deploy

type: git
repo: 仓库地址

步骤3： 安装hexo-deployer-git自动部署发布工具，指令:npm install hexo-deployer-git --save
步骤4： 通过命令hexo clean && hexo g && hexo d发布到GitHub
最后输入https://hanzawa1205.github.io测试访问部署完成的博客。
